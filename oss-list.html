My Open Source Wishlist

Want to get started contributing to open source? Or need some inspiration on what to take up next? I’ve got you covered! This page is an unordered list of feature requests and ideas that I’m excited about. However, just like you, my time is finite…

Feel free to take an item of this list and start working on it. Don’t hesitate to message me about any of the items on this list. I would like to brainstorm and help you contribute them (also if this is your first contribution!).

You can reach out to me on Symfony Slack as @wouterj, at Mastodon or directly on GitHub.

  • Share something you've learned this week Each week you write code, you learn something. If you think it's easy, it's because you've mastered that something. Consider sharing how to do the things you think are easy, this will massively help people in your community!
  • Add alt texts to images in symfony/symfony-docs Let's improve accessibility of the Symfony documentation by adding relevant alt descriptions to our figures.
  • Add (de)authentication information to the profiler in symfony/symfony
  • Add "sudo mode" in symfony/symfony The goal is to add "authentication factors"/"level of assurance" support in Symfony, to tackle both MFA and sudo mode in one feature. There is a GitHub gist with some more details.
  • Integrate Symfony Console Completion in symfony-cli/symfony-cli If we add shell completion to Symfony CLI, we can automatically integrate the Symfony shell completion for symfony console ...
  • Add ref role implementation in phpDocumentor/guides The new reStructuredText parser currently does not support referencing other sections in the docs. This is one of the major missing features before using it in the Symfony documentation.
